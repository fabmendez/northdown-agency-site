# Marketing CTA Improvement Plan

## Purpose

This plan identifies how to make the public site more buyer-facing while preserving Northdown's approved positioning as an Operational Systems Engineering company.

It is a planning document only. It does not approve implementation, dependencies, analytics, deployment, contact backend work, proof claims, case studies, metrics, or legal guarantees.

## Positioning Guardrails

- Keep the category: Operational Systems Engineering.
- Do not position Northdown as an AI agency, generic automation company, or prompt consultancy.
- Do not invent proof, metrics, testimonials, case studies, certifications, or compliance claims.
- Keep language credible, commercial, operational, restrained, buyer-facing, and specific.
- Use AI language only where it supports operational capability.
- Keep discovery, validation, governance, and human oversight visible.

## Global CTA Direction

### Current weakness

CTA styling is mostly consistent, but the site does not yet use CTAs as a clear conversion system. Some secondary CTAs use different language for similar actions, and some pages describe next steps without making the buyer's immediate path feel concrete.

### Recommended improvement

- Standardise primary CTA language around `Request Operational Review` for the contact path.
- Use `Book Discovery` where the surrounding page is already framing a discovery conversation.
- Use the approved public discovery phone number as a secondary/supporting action: `020-3773 0511`.
- Use secondary CTAs to route visitors to the next best page: `See Services`, `See Process`, or `Review Privacy Notes`.
- Add short CTA support copy that explains what the action is for.
- Keep CTA blocks visually intentional and consistent across pages.
- Do not imply a working contact form or backend submission path.

### Implementation timing

Implement now across public marketing pages except Privacy, where CTA treatment should remain minimal.

## Homepage

### Current weakness

The homepage has the right headline and positioning, but the flow still reads partly like an operating model. It explains what Northdown believes before it fully sharpens buyer recognition, urgency, and the cost of leaving operational drag unresolved.

### Buyer intent

The visitor is trying to decide whether Northdown understands their operational problem and whether it is worth continuing to Services, Process, or Contact.

### Recommended marketing improvement

- Strengthen the "why now" section around this message: growth should not make the business harder to run.
- Move from problem labels toward recognition copy that mirrors buyer pain: repeated work, missing ownership, knowledge loss, inconsistent delivery, and staff dependency.
- Make the operating model feel like a buyer benefit, not just an internal method.
- Reframe service overview around outcomes: less repeated admin, retained knowledge, clearer workflows, more consistent delivery.
- Keep trust content focused on governance, validation, and human oversight.

### Recommended CTA improvement

- Primary CTA: `Book Discovery`.
- Secondary CTA: `See Services`.
- Replace `See Systems` with `See Services` for clarity and consistency.
- Add support copy that says discovery starts with the workflow before tools or implementation choices.

### Implementation timing

Implement now. Homepage is the highest-leverage page for buyer recognition and conversion clarity.

## Services

### Current weakness

The Services page explains categories clearly, but the service cards still read like labels and definitions. Buyers need each service to answer what problem it solves, what Northdown builds, and what operational outcome improves.

### Buyer intent

The visitor wants to understand whether Northdown offers the kind of help they need and which service path fits their operational problem.

### Recommended marketing improvement

- Expand each service card into a buyer-facing structure:
  - problem it solves
  - what Northdown builds
  - outcome that improves
- Keep the four current public service categories:
  - Operational AI Systems
  - Workflow Automation
  - Knowledge Systems
  - Operational Infrastructure
- Avoid turning service cards into packages, pricing, timelines, or implementation promises.
- Make the good-fit section feel more commercial by connecting fit criteria to readiness for discovery.

### Recommended CTA improvement

- Primary CTA: `Request Operational Review`.
- Secondary CTA: `See Process`.
- CTA copy should state that the review identifies where systems would create leverage before implementation decisions.

### Implementation timing

Implement now. Services is the main offer page and needs clearer commercial value.

## Process

### Current weakness

The Process page is accurate, but it is still written like a delivery checklist. It should reduce buyer anxiety by showing controlled, low-chaos delivery and clear decision points.

### Buyer intent

The visitor wants to know whether Northdown's work will be controlled, practical, and safe enough to trust with operational workflows.

### Recommended marketing improvement

- Frame the process around risk reduction:
  - understand first
  - scope before build
  - validate before handoff
  - preserve human approval
  - avoid tool-first mistakes
- Keep the existing steps, but make each one show buyer value and control.
- Add a short "what this prevents" angle: unclear scope, uncontrolled automation, poor handoff, and systems nobody owns.

### Recommended CTA improvement

- Primary CTA: `Request Operational Review`.
- Secondary CTA: `See Services`.
- CTA copy should invite the visitor to start with a review of the current workflow.

### Implementation timing

Implement now after Services. Process is the strongest trust-building page for qualified buyers.

## About

### Current weakness

The About page explains Northdown's philosophy, but it could build more buyer confidence by translating beliefs into practical delivery controls.

### Buyer intent

The visitor wants confidence that Northdown is credible, disciplined, and careful with operational work.

### Recommended marketing improvement

- Reduce internal philosophy language where possible.
- Emphasise buyer trust signals:
  - governed delivery
  - validation before handoff
  - clear boundaries
  - practical systems over tool-chasing
  - human oversight
- Avoid founder biography, credentials, history, or client claims unless approved evidence is provided.
- Keep "what Northdown is not" but connect it to buyer protection: no hype-led implementation, no tool-first push, no uncontrolled autonomy.

### Recommended CTA improvement

- Primary CTA: `Book Discovery`.
- Secondary CTA: `See Process`.
- CTA copy should say the next step is to review the operation before choosing a system.

### Implementation timing

Implement after Homepage, Services, and Process. It is important, but should support the conversion path rather than lead it.

## Resources

### Current weakness

The Resources page is transparent, but the resource library status can make the page feel empty. It should provide useful decision support now without pretending published resources exist.

### Buyer intent

The visitor is researching operational AI, automation, or systems and wants to self-diagnose whether their business needs better operational infrastructure.

### Recommended marketing improvement

- Keep the transparent resource status, but move it lower on the page.
- Add stronger decision-support sections:
  - questions to diagnose operational drag
  - signs you need systems before more hires
  - what should stay human
  - when to start with discovery instead of tools
- Route visitors clearly to Services, Process, and Contact.
- Do not invent guides, articles, downloads, case studies, or FAQs unless separately approved.

### Recommended CTA improvement

- Primary CTA: `See Services` for education-stage visitors.
- Secondary CTA: `Book Discovery` for visitors ready to act.
- Add a mid-page route card for `Request Operational Review` if implementation remains clear that the contact method is pending.

### Implementation timing

Implement later, after the main conversion pages. Resources can become stronger once the core buyer journey is improved.

## Contact

### Current weakness

Contact is clear but not conversion-focused enough. Without a backend form, the page needs to make the pending action obvious while still making discovery feel worthwhile.

### Buyer intent

The visitor is close to taking action and needs to know what discovery is for, what to prepare, and what not to send yet.

### Recommended marketing improvement

- Put the conversion action closer to the top of the page.
- Strengthen the explanation of discovery:
  - identify the operational problem
  - understand current workflows
  - assess fit
  - decide whether an audit or proposal is appropriate
- Add a concise "good enquiry includes" section based on the intake template.
- Keep privacy/access warnings plain and visible.
- Do not imply a working form, response-time guarantee, CRM, email route, or automation.

### Recommended CTA improvement

- Primary CTA block headline: `Request Operational Review`.
- Approved public discovery contact method: `020-3773 0511`.
- Primary visual CTA: `Request Operational Review`.
- Secondary CTA: `Call 020-3773 0511`.
- Supporting link/button: `Review Privacy Notes`.
- Keep phone positioned as a direct enquiry option, not the main conversion action.
- Include honest implementation copy: `A fuller contact or booking flow is planned. For now, use the available contact options below.`
- Contact no longer needs `Contact method pending final configuration` placeholder language.
- Do not add or imply a working contact form, booking system, CRM, email route, response-time promise, or automation.

### Implementation timing

Implement now using the approved phone number. This is the main conversion endpoint.

## Privacy

### Current weakness

The Privacy page is appropriately cautious. It does not need stronger marketing treatment, but the CTA and contact status can be less visually dominant than marketing pages.

### Buyer intent

The visitor wants assurance about sensitive information before contacting Northdown.

### Recommended marketing improvement

- Keep plain-language privacy guidance.
- Keep the current site status visible.
- Avoid adding persuasion, urgency, or commercial language.
- Consider adding a short "before sending information" note that reinforces not sending credentials or private data casually.

### Recommended CTA improvement

- Keep CTA minimal.
- Use `Return To Contact` or `See Services`.
- Do not use aggressive conversion copy.

### Implementation timing

Later, unless legal/privacy review requests changes. Avoid over-marketing this page.

## Recommended Implementation Order

1. Homepage CTA and "why now" refinement.
2. Contact conversion path refinement, while preserving placeholder contact action.
3. Services outcome-led card rewrite.
4. Process risk-reduction rewrite.
5. About trust-confidence refinement.
6. Resources decision-support refinement.
7. Privacy minimal review only.

## Validation Approach For Future Implementation

For each implementation slice:

- Change one page or one closely related CTA group at a time.
- Update matching `content/pages/` markdown when public copy changes.
- Run `npm.cmd run build`.
- Run `npm.cmd run check`.
- Confirm no unapproved route, dependency, Docker, analytics, contact backend, or deployment changes were introduced.
- Review copy against this plan and the approved project brain guardrails.

## Assumptions

- The current page content is valid first-pass public content, but it needs stronger buyer-facing flow.
- The approved current public discovery contact method is phone: `020-3773 0511`.
- Phone is secondary/supporting to `Request Operational Review` in the CTA hierarchy.
- The contact form backend remains unimplemented.
- No proof assets, case studies, testimonials, pricing, or metrics are approved yet.
- Future implementation should preserve the current visual restraint while improving hierarchy, section order, and CTA clarity.

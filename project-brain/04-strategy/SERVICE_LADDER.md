# Service Ladder

## Purpose

The service ladder gives future site content and sales conversations a clear progression. It should help a business start with a low-risk review, then move into implementation and ongoing improvement when there is a practical case.

## 1. Operational AI Readiness Review

A focused review of current workflows, repeated tasks, systems, data sources, and risks.

Typical outputs:

- Workflow map.
- Automation opportunities list.
- Risk and access notes.
- Recommended first project.
- Clear next-step estimate or scope.

Best for businesses that know operations are inefficient but are not sure where AI or automation should start.

## 2. Workflow Automation Sprint

A small implementation project for one defined workflow.

Typical outputs:

- Approved workflow design.
- Working automation or internal process aid.
- Human review points.
- Basic documentation.
- Validation checklist.

Best for repeated tasks with clear inputs, outputs, and ownership.

## 3. Internal Tool Build

A more substantial project to create a business-specific tool or operating base.

Typical outputs:

- Internal dashboard or workflow tool.
- Role-based process views.
- Data handling rules.
- Documentation for staff use.
- Maintenance and improvement plan.

Best for businesses outgrowing spreadsheets, inboxes, or disconnected tools.

## 4. Agentic Engineering System

A structured AI-assisted system where agents, tools, human review, and source-of-truth documents work together.

Typical outputs:

- Agent operating contract.
- Project-brain or knowledge base structure.
- Tool and approval boundaries.
- Audit-friendly workflow design.
- Validation and handoff procedures.

Best for teams that need repeatable AI-assisted work without losing control or traceability.

## 5. Ongoing Operations Support

Support for improving, maintaining, and safely extending systems after launch.

Typical outputs:

- Regular review cycles.
- Workflow improvements.
- Documentation updates.
- Security and dependency checks.
- New automation backlog management.

Best for businesses that want systems to keep improving without unmanaged sprawl.
